## 1. Executive Summary

### 1.1 제품 개요

**한 줄 요약**: 방탈출 카페에서 플레이어들에게 실시간으로 힌트를 제공하는 디지털 힌트 관리 시스템

**제품 설명**:

EscapeHint는 방탈출 카페의 힌트 제공 프로세스를 디지털화하여, 플레이어는 게임 몰입도를 유지하면서 필요한 힌트를 즉시 확인할 수 있고, 관리자는 힌트를 효율적으로 관리할 수 있는 웹 기반 솔루션입니다.

**핵심 가치**:
- 플레이어: 게임 몰입도 유지하며 즉시 힌트 확인
- 관리자: 힌트 관리 시간 절약 및 유연한 수정
- 카페 운영자: 운영 효율성 향상 및 고객 만족도 증대

### 1.2 핵심 문제 정의

**현재 방탈출 카페가 겪는 문제**:

1. **종이 힌트의 한계**
   - 분실 위험이 높고 물리적 관리가 어려움
   - 힌트 내용 수정 시 재인쇄 필요
   - 오래된 힌트는 훼손되거나 가독성 저하

2. **직원 호출 방식의 문제**
   - 플레이어의 게임 몰입도 저하
   - 직원의 대응 시간 지연 가능
   - 동시 다발적 요청 처리 어려움

**해결 방안**:

EscapeHint는 디지털 힌트 시스템으로 이러한 문제를 해결하며, 다음 조건만 충족하면 즉시 도입 가능합니다:
- PC 또는 스마트폰 보유
- 안정적인 Wi-Fi 환경

## 2. Product Overview

### 2.1 핵심 가치 제안

**플레이어(Customer) 관점**:

- 직관적인 UI로 게임 몰입도를 유지하면서 필요한 힌트 즉시 확인
- 어두운 방에서도 읽기 쉬운 대형 폰트와 다크 모드
- 힌트 코드만 입력하면 즉시 힌트 확인 (3클릭 이내)

**관리자(Admin) 관점**:

- 테마별 힌트를 5분 내 손쉽게 등록/수정
- 비공개 관리 인터페이스로 플레이어에게 노출 방지
- 힌트 순서 조정 및 활성화/비활성화 제어

**운영(Operations) 관점**:

- 게임 진행률 실시간 추적
- 힌트 사용 통계 확인으로 난이도 조정 가능
- 직원 시간 절약 (월 평균 XX시간)

### 2.2 주요 기능 개요

1. **플레이어 기능**: 테마 선택 → 게임 시작 → 힌트 코드 입력 → 힌트/정답 확인
2. **관리자 기능**: 관리자 모드 진입 → 테마 관리 → 힌트 관리 → 세션 모니터링
3. **공통 기능**: 실시간 타이머, 진행률 표시, 세션 관리

## 3. User Personas

### 3.1 플레이어 페르소나

#### 페르소나 1: 초보 플레이어 "민수"

- **연령대**: 10~20대 중반
- **방탈출 경험**: 5회 미만
- **기대 경험**: 친구들과 즐거운 시간, 성취감
- **힌트 태도**: 막히면 빨리 힌트를 보고 싶어함
- **디지털 리터러시**: 높음, 스마트폰 앱 사용에 익숙
- **주요 니즈**: "게임 중 힌트를 빠르게 받아서 편하게 계속 진행하고 싶다"
- **사용 패턴**: 평균 3-5개 힌트 사용, 20분마다 1개 힌트 확인

#### 페르소나 2: 고급 플레이어 "지은"

- **연령대**: 20대 중반
- **방탈출 경험**: 20회 이상
- **기대 경험**: 높은 난이도 도전, 스스로 해결하는 성취감
- **힌트 태도**: 최대한 힌트 안 보려 함
- **디지털 리터러시**: 높음
- **주요 니즈**: "힌트 없이 해결하되, 정말 필요할 때만 정확한 힌트를 받고 싶다"
- **사용 패턴**: 평균 0-2개 힌트 사용, 막힐 때만 선택적으로 확인

### 3.2 관리자 페르소나

#### 페르소나 3: 카페 사장님 "현우"

- **역할**: 방탈출 카페 운영자
- **IT 숙련도**: 중간 (비전문가, 웹 브라우저 사용 가능)
- **하루 평균 관리 시간**: 10분 이내 목표
- **주요 불편사항**: 종이 힌트 정리, 힌트 변경 시 재인쇄, 훼손된 힌트 교체
- **주요 니즈**: "새 테마를 빠르게 등록하고, 고객 피드백에 따라 힌트를 쉽게 수정하고 싶다"
- **사용 패턴**: 주 1-2회 테마 업데이트, 주 3-4회 힌트 수정

## 4. User Stories

### 4.1 플레이어 스토리

#### 필수 스토리 (P0)

**US-01: 테마 선택 및 게임 시작**

> 플레이어로서, 테마를 선택하고 게임을 시작하여 타이머가 작동하는 것을 보고 싶다. 왜냐하면 제한 시간을 인지하고 긴장감을 느끼고 싶기 때문이다.

- **참조**: 도메인 정의서 UC-01
- **AC (Acceptance Criteria)**:
  - 활성화된 테마 목록이 카드 형태로 표시됨
  - 테마 선택 시 게임 세션이 자동 생성됨
  - 타이머가 자동으로 시작되고 카운트다운됨
  - 세션 ID가 브라우저에 저장되어 새로고침 시에도 유지됨

**US-02: 힌트 코드 입력 및 조회**

> 플레이어로서, 힌트 코드를 빠르게 입력하고 힌트 내용을 확인하고 싶다. 왜냐하면 게임에 집중하고 싶기 때문이다.

- **참조**: 도메인 정의서 UC-02
- **AC**:
  - 힌트 코드 입력 필드가 화면 중앙에 크게 표시됨
  - 코드 입력 후 엔터 또는 버튼 클릭으로 제출
  - 유효한 코드는 힌트 내용 + 진행률 + 정답보기 버튼 표시
  - 무효한 코드는 명확한 오류 메시지 표시 (예: "힌트를 찾을 수 없습니다")
  - 대소문자 구분 없이 코드 인식

**US-03: 정답 확인**

> 플레이어로서, 정답보기 버튼을 눌러 정답을 확인하고 싶다. 왜냐하면 힌트를 봐도 모르겠을 때 정답을 알고 넘어가고 싶기 때문이다.

- **참조**: 도메인 정의서 UC-03
- **AC**:
  - 정답보기 버튼 클릭 시 정답 텍스트가 명확하게 표시됨
  - 정답은 큰 글자 크기로 읽기 쉽게 표시
  - 정답 확인 후에도 힌트 내용은 계속 볼 수 있음

**US-04: 플레이 정보 확인**

> 플레이어로서, 현재 플레이 정보(남은 시간, 사용한 힌트 수, 진행률)를 항상 확인하고 싶다. 왜냐하면 게임 진행 상황을 파악하고 싶기 때문이다.

- **참조**: 도메인 정의서 UC-04
- **AC**:
  - 타이머가 화면 상단에 실시간으로 갱신됨 (MM:SS 형식)
  - 사용한 힌트 수가 정확히 표시됨 (예: "힌트 3/12 사용")
  - 현재 진행률이 퍼센트로 표시됨 (예: "진행률: 45%")
  - 모든 정보가 한 화면에서 확인 가능

#### 중요 스토리 (P1)

**US-04-1: 세션 복구**

> 플레이어로서, 브라우저를 실수로 닫거나 새로고침해도 게임 세션이 유지되길 원한다. 왜냐하면 게임 진행 상황을 잃고 싶지 않기 때문이다.

- **AC**:
  - 브라우저 새로고침 시 세션 정보 유지
  - 사용했던 힌트 목록 유지
  - 타이머는 서버 시간 기준으로 정확하게 복구

### 4.2 관리자 스토리

#### 필수 스토리 (P0)

**US-05: 관리자 모드 진입**

> 관리자로서, 비밀스럽게 관리자 모드에 진입하고 싶다. 왜냐하면 플레이어에게 노출되지 않아야 하기 때문이다.

- **참조**: 도메인 정의서 UC-05
- **AC**:
  - 플레이어 화면의 숨겨진 영역(예: 하단 로고)을 5회 클릭 시 비밀번호 입력 화면 표시
  - 비밀번호 입력 후 관리자 대시보드로 이동
  - 관리자 세션은 2시간 유지 (JWT 토큰 기반)
  - 관리자 모드 진입 방법은 플레이어에게 보이지 않음

**US-06: 테마 관리**

> 관리자로서, 테마를 5분 안에 등록하고 싶다. 왜냐하면 바쁜 운영 시간에 빠르게 처리해야 하기 때문이다.

- **참조**: 도메인 정의서 UC-06
- **AC**:
  - 테마 이름, 설명, 제한시간 입력 후 저장
  - 중복 테마 이름 검증 (실시간 피드백)
  - 활성화/비활성화 토글로 간편하게 설정
  - 테마 목록에서 수정/삭제 가능
  - 삭제 시 확인 메시지 표시 (실수 방지)

**US-07: 힌트 관리**

> 관리자로서, 힌트를 쉽게 등록하고 순서를 조정하고 싶다. 왜냐하면 힌트는 자주 수정되기 때문이다.

- **참조**: 도메인 정의서 UC-07
- **AC**:
  - 힌트 코드, 내용, 정답, 진행률 입력 폼 제공
  - 힌트 순서 조정 (위/아래 버튼 또는 드래그앤드롭)
  - 힌트 활성화/비활성화 토글
  - 힌트 미리보기 기능 (플레이어 화면 시뮬레이션)
  - 대량 등록 시 템플릿 다운로드 기능 (CSV)

#### 중요 스토리 (P1)

**US-08: 세션 모니터링**

> 관리자로서, 진행 중인 게임 세션을 모니터링하고 싶다. 왜냐하면 고객 지원이 필요할 수 있기 때문이다.

- **참조**: 도메인 정의서 UC-08
- **AC**:
  - 진행 중인 세션 목록 실시간 조회 (테마명, 남은 시간, 사용 힌트 수)
  - 세션별 사용 힌트 현황 상세 확인
  - 세션 강제 종료 가능 (게임 재시작 시)
  - 세션 통계 요약 (평균 힌트 사용 수, 평균 완료 시간)

### 5.2 기능 상세 명세

#### F-01: 테마 선택 및 게임 시작 (P0)

- **설명**: 플레이어가 활성 테마 목록을 보고 선택하여 게임 세션을 시작
- **입력**: 테마 ID
- **출력**: 게임 세션 ID, 타이머 시작
- **처리 흐름**:
  1. 활성화된 테마 목록 조회 (GET /api/themes?status=active)
  2. 플레이어가 테마 선택
  3. 게임 세션 생성 (POST /api/sessions)
  4. 세션 ID를 localStorage에 저장
  5. 타이머 시작 (서버 시간 기준)
- **비즈니스 규칙**: BR-03 (세션은 하나의 테마만 플레이)
- **예외 처리**:
  - 비활성 테마는 목록에 표시 안 함
  - 네트워크 오류 시 로컬 캐시 활용
  - 이미 진행 중인 세션이 있으면 복구 여부 확인

#### F-02: 힌트 코드 입력 및 조회 (P0)

- **설명**: 플레이어가 힌트 코드를 입력하여 힌트 내용 확인
- **입력**: 힌트 코드 (대소문자 구분 없음)
- **출력**: 힌트 내용, 진행률, 정답보기 버튼
- **처리 흐름**:
  1. 플레이어가 코드 입력 후 제출
  2. 코드 검증 (POST /api/sessions/:id/hints)
  3. 유효한 코드인 경우 힌트 내용 표시
  4. 세션에 사용 힌트 기록
  5. 진행률 자동 계산 및 업데이트
- **비즈니스 규칙**:
  - BR-01 (테마 내 코드 고유성)
  - BR-04 (동일 코드 재입력 시 카운트 증가 안 함)
  - BR-06 (코드 검증 실패 처리)
- **예외 처리**:
  - 존재하지 않는 코드: "힌트를 찾을 수 없습니다"
  - 비활성 힌트: "현재 사용할 수 없는 힌트입니다"
  - 다른 테마 코드: "현재 테마에서 사용할 수 없는 코드입니다"

#### F-03: 실시간 타이머 (P0)

- **설명**: 게임 시작부터 남은 시간 실시간 카운트다운
- **입력**: 세션 시작 시간, 테마 제한 시간
- **출력**: MM:SS 형식의 남은 시간
- **처리 흐름**:
  1. 세션 시작 시 서버 시간 기준 시작 시각 기록
  2. 클라이언트에서 1초마다 남은 시간 계산
  3. 60초마다 서버 시간과 동기화 (타이머 정확도 유지)
  4. 탭 전환 후 복귀 시 서버 시간 기준으로 재계산
  5. 시간 종료 시 알림 표시
- **비즈니스 규칙**: BR-05 (타이머는 백그라운드에서도 계속 작동)
- **예외 처리**:
  - 네트워크 단절 시 로컬 시간으로 계속 작동
  - 재연결 시 서버 시간과 동기화

#### F-04: 테마 관리 (P0)

- **설명**: 관리자가 테마를 추가, 수정, 삭제
- **입력**: 테마 이름, 설명, 제한시간, 활성화 여부
- **출력**: 테마 목록, 생성/수정/삭제 성공 메시지
- **처리 흐름**:
  - **생성**: POST /api/admin/themes
  - **조회**: GET /api/admin/themes
  - **수정**: PUT /api/admin/themes/:id
  - **삭제**: DELETE /api/admin/themes/:id
- **비즈니스 규칙**: BR-08 (테마 삭제 시 관련 힌트도 모두 삭제)
- **예외 처리**:
  - 중복 테마 이름 검증
  - 진행 중인 세션이 있는 테마는 삭제 불가 (경고 표시)

#### F-05: 힌트 관리 (P0)

- **설명**: 관리자가 힌트를 추가, 수정, 삭제, 순서 조정
- **입력**: 힌트 코드, 내용, 정답, 진행률, 순서
- **출력**: 힌트 목록, 생성/수정/삭제 성공 메시지
- **처리 흐름**:
  - **생성**: POST /api/admin/themes/:themeId/hints
  - **조회**: GET /api/admin/themes/:themeId/hints
  - **수정**: PUT /api/admin/hints/:id
  - **삭제**: DELETE /api/admin/hints/:id
  - **순서 조정**: PATCH /api/admin/hints/:id/order
- **비즈니스 규칙**: BR-01 (테마 내 힌트 코드 고유성)
- **예외 처리**:
  - 중복 힌트 코드 검증
  - 진행률은 0-100% 범위 검증

#### F-06: 관리자 인증 (P0)

- **설명**: 관리자 로그인 및 세션 관리
- **입력**: 비밀번호
- **출력**: JWT Access Token, Refresh Token
- **처리 흐름**:
  1. 플레이어 화면에서 숨겨진 버튼 5회 클릭
  2. 비밀번호 입력 화면 표시
  3. 비밀번호 검증 (POST /api/admin/auth/login)
  4. 성공 시 JWT 토큰 발급 및 HttpOnly 쿠키에 저장
  5. 관리자 대시보드로 리다이렉트
- **비즈니스 규칙**: BR-09 (관리자 비밀번호는 bcrypt로 해싱)
- **예외 처리**:
  - 3회 이상 실패 시 5분 잠금
  - 토큰 만료 시 Refresh Token으로 자동 갱신

   - 세션 상세 보기
   - 강제 종료 버튼

## 9. Business Rules

### 9.1 핵심 비즈니스 규칙

도메인 정의서의 BR-01~BR-09를 그대로 적용하며, 다음과 같이 요약합니다:

**BR-01: 힌트 코드 고유성**
- 하나의 테마 내에서 힌트 코드는 중복될 수 없음
- 대소문자 구분 없이 고유성 검증

**BR-02: 진행률 자동 계산**
- 힌트 사용 시 해당 힌트의 진행률을 세션에 자동 적용
- 여러 힌트 사용 시 가장 높은 진행률로 갱신

**BR-03: 세션-테마 관계**
- 하나의 게임 세션은 하나의 테마만 플레이
- 세션 진행 중 테마 변경 불가

**BR-04: 힌트 중복 사용 처리**
- 동일한 힌트 코드를 재입력해도 사용 횟수는 증가하지 않음
- 이미 본 힌트는 자유롭게 재확인 가능

**BR-05: 타이머 백그라운드 작동**
- 타이머는 백그라운드에서도 계속 작동
- 탭 전환 후 복귀 시 서버 시간 기준으로 정확히 표시

**BR-06: 힌트 코드 검증**
- 존재하지 않는 코드, 비활성 힌트, 다른 테마 코드는 각각 다른 오류 메시지 표시

**BR-07: 세션 복구**
- 브라우저 새로고침 시 localStorage의 세션 ID로 세션 복구
- 서버 세션과 동기화하여 정확한 상태 유지

**BR-08: 테마 삭제 제약**
- 테마 삭제 시 관련된 모든 힌트도 함께 삭제 (Cascade Delete)
- 진행 중인 세션이 있는 테마는 삭제 불가 (경고 메시지)

**BR-09: 보안 규칙**
- 관리자 비밀번호는 bcrypt로 해싱하여 저장
- JWT 토큰은 HttpOnly 쿠키에 저장하여 XSS 방지

### 16.2 용어 사전

| 용어      | 영문                   | 정의                                       |
| --------- | ---------------------- | ------------------------------------------ |
| 테마      | Theme                  | 방탈출 게임의 하나의 콘텐츠 단위           |
| 힌트      | Hint                   | 게임 진행을 돕기 위한 단서 정보            |
| 힌트 코드 | Hint Code              | 힌트에 접근하기 위해 입력하는 고유 코드    |
| 진행률    | Progress Rate          | 해당 힌트를 받았을 때의 게임 진행 정도 (%) |
| 세션      | Game Session           | 하나의 팀이 하나의 테마를 플레이하는 단위  |
| MVP       | Minimum Viable Product | 최소 기능 제품                             |
| P0/P1/P2  | Priority 0/1/2         | 기능 우선순위 (0: 필수, 1: 중요, 2: 선택)  |
| NPS       | Net Promoter Score     | 고객 충성도 지표                           |
| JWT       | JSON Web Token         | 토큰 기반 인증 방식                        |
| CRUD      | Create/Read/Update/Delete | 데이터 생성/조회/수정/삭제 기본 작업      |

### 16.3 약어 및 용어 설명

- **API**: Application Programming Interface - 애플리케이션 간 통신 인터페이스
- **REST**: Representational State Transfer - RESTful API 설계 원칙
- **HTTPS**: HyperText Transfer Protocol Secure - 암호화된 HTTP 통신
- **XSS**: Cross-Site Scripting - 악성 스크립트 삽입 공격
- **CSRF**: Cross-Site Request Forgery - 교차 사이트 요청 위조
- **P95**: 95th Percentile - 상위 5%를 제외한 95%의 응답 시간
- **E2E**: End-to-End - 전체 시나리오 통합 테스트
